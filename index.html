Release Early, Release Often: Predicting Change in Versioned KOS on the Web

A companion website for the paper Release Early, Release Often: Predicting Change in Versioned Knowledge Organization Systems on the Web.

This page contains additional data and analysis supporting the paper:

  • Characteristics of input datasets
  • Selected features per dataset
  • Finding relationships between features and classifier performance
  • Finding relationships between features and best classifier

The source code of the paper's approach is located here. The 139 KOS version chains used as input data can be found here. Scripts with statistical analyses (i.e. regressions) can be found here.

Characteristics of input datasets

The following table describes the 139 datasets evalauted in the paper. Each dataset consists of exactly one KOS version chain. One version chain consists of a varying number of KOS versions (at least 3). Datasets 1 and 2 are the CEDAR and DBpedia datasets. Datasets 3-136 are the LOV datasets (i.e. retrieved via the Linked Open Vocabularies API). Datasets 137-139 are the SPARQL datasets (i.e. KOS version chains reconstructed after querying the 637 public SPARQL endpoints in datahub.io). Each dataset is described by the following characteristics/features:

  • dataset: name of the version chain
  • totalSize: total size of the version chain, adding up the total number of triples in all versions
  • nSnapshots: number of versions/snapshots
  • avgGap: average time gap between the versions (in number of days) considering their release date
  • avgSize: average size of the version chain, in number of triples
  • nInserts: total number of inserted triples between all version pairs
  • nDeletes: total number of deleted triples between all version pairs
  • nComm: total number of common/stable triples between all version pairs
  • ratioInserts: ratio of inter-version statement insertions versus total number of inter-version operations
  • ratioDeletes: ratio of inter-version statement deletions versus total number of inter-version operations
  • ratioComm: ratio of inter-version common statements versus total number of inter-version operations
  • isTree: whether the KOS forms a tree or a graph
  • maxTreeDepth: maximum tree depth considering all versions
  • avgTreeDepth: average tree depth
  • totalInstances: total number of instances in the version chain
  • ratioInstances: ratio of instance triples versus total number of triples
  • ratioInstancesVSIS: ratio of instance triples versus instance and structural triples
  • totalStructural: total number of structural properties in the version chain
  • ratioStructural: ratio of structural triples versus total number of triples
  • ratioStructuralVSIS: ratio of structural triples versus instance and structural triples
